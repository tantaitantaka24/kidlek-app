# โปรเจกต์: คิดเลขเร็ว — โปรแกรมฝึกซ้อมแข่งศิลปหัตถกรรม

## ภาษาที่ใช้คุย

- ตอบเป็นภาษาไทยเป็นหลัก (คำเทคนิคใช้ภาษาอังกฤษได้)
- ตอบตรงประเด็นทันที ไม่ต้องเกริ่น
- ถ้าไม่แน่ใจให้บอกตรง ๆ ห้ามเดา
- ถ้ามีหลายแนวทาง เสนอ 2-3 ทางก่อนลงมือทำใหญ่
- เวลาแก้งาน แก้เฉพาะที่ขอ ห้ามปรับปรุงจุดอื่นโดยไม่ถาม

## บริบทของโปรเจกต์

โปรแกรมฝึกซ้อมการแข่งคิดเลขเร็ว ศิลปหัตถกรรมนักเรียน

### ระดับที่รองรับ

- ป.1-3: operations + − × ÷ ^
- ป.4-6: เพิ่ม √ และ ⁿ√ (รากอันดับ n)

### โจทย์

- 4 ตัวเลข → ผลลัพธ์ 2 หลัก (10-99)
- 5 ตัวเลข → ผลลัพธ์ 3 หลัก (100-999)
- ตัวเลขโจทย์ ห้ามซ้ำเกิน 2 ตัว, เลข 0 มีได้ไม่เกิน 1 ตัว

### กฎ Operation

- **ทุก operation รับ sub-expression เป็น operand ได้** (เช่น 9^(1+1), √(4^8))
- ใช้ตัวเลขให้ครบทุกตัว ตัวละครั้ง
- ค่ากลางติดลบและเศษส่วนได้
- ผลสุดท้ายต้องเป็นจำนวนเต็มบวก (ตามจำนวนหลักที่กำหนด หรือใกล้เคียงสุด)
- ราก: ผลต้องเป็นจำนวนเต็ม, รากอันดับ 2 ใช้ √ ชั้นเดียว, ห้ามรากอนันต์
- 0^0 ห้าม, 5^0=1, 5^1=5 ใช้ได้

## Tech Stack

- HTML ไฟล์เดียว + vanilla JS + Web Worker
- ไม่มี build step, เปิดด้วย double-click ได้
- Solver: DP over bitmask, dedupe by canonical form, timeout 5 วินาที

## โครงสร้างไฟล์

- `คิดเลขเร็ว.html` — ไฟล์หลักทั้งหมด (~2,200 บรรทัด)
  - บรรทัด 1-770: HTML + CSS
  - บรรทัด 771-1700: Solver code ใน <script id="solver-worker">
  - บรรทัด 1700-end: Main UI logic

## Performance ปัจจุบัน

| Case        | Avg   | Max    |
| ----------- | ----- | ------ |
| ป.1-3 4 ตัว | 6ms   | 10ms   |
| ป.4-6 5 ตัว | 350ms | ~800ms |

## ฟีเจอร์ที่ทำเสร็จแล้ว

- เลือกระดับ → ตั้งค่า → เล่น → ดูเฉลย/ใบ้
- Timer พร้อม option ไม่จับเวลา และ bip 5 วิสุดท้าย
- เฉลย 2-3 วิธีต่างกันจริง (ไม่ใช่แค่สลับตำแหน่ง)
- ปุ่มขอใบ้ก่อนดูเฉลย
- LocalStorage เก็บสถิติ
- โหมดพิมพ์ 10/20/30 ข้อ พร้อมเฉลย

## จุดที่อาจต้องปรับ (ยังไม่ทำ)

1. Hint quality — ตอนนี้แค่บอกขั้นแรกของวิธีที่ 1
2. Mobile UI — ยังไม่ทดสอบบนจอเล็กละเอียด
3. บางครั้ง solver หา exact ไม่เจอเพราะ cap 30,000 ที่ fullMask

## วิธีทดสอบ

ไม่มี automated test — เปิดไฟล์ในเบราว์เซอร์โดยตรง
ถ้าจะทดสอบ solver แยก ใช้ Node.js extract <script id="solver-worker"> มา eval แล้วเรียก self.onmessage
